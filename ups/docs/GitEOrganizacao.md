# Git e Organização do Projeto

## Introdução

Nesta secção é descrita a forma como o **Git** foi utilizado no desenvolvimento do projeto, bem como as **boas práticas de organização** adotadas ao longo do trabalho.

O uso correto do controlo de versões foi um dos objetivos do projeto, permitindo acompanhar a evolução do código, facilitar a colaboração entre os elementos do grupo e garantir a segurança do trabalho desenvolvido.

---

## Controlo de Versões com Git

O **Git** foi utilizado como sistema de controlo de versões para gerir todas as alterações efetuadas no projeto ao longo do semestre.

A utilização do Git permitiu:
- Registar o histórico de alterações
- Reverter alterações quando necessário
- Facilitar o trabalho em grupo
- Garantir a integridade do código

---

## Estrutura do Repositório

O repositório encontra-se organizado de forma clara, separando:

- Código fonte da aplicação
- Dados (ficheiros JSON)
- Documentação do projeto

Esta separação contribui para um repositório mais limpo e de fácil compreensão.

---
